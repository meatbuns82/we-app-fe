# 前端美化技能 (Frontend Beautification Skill)

## 参考标准
Awwwards


## 技能目标
生成视觉优雅、交互流畅、响应式且符合现代设计趋势的前端界面代码。适用于网页、组件库、落地页、后台系统等多种场景。

## 核心设计原则
1. **视觉层次** – 使用字体大小、粗细、颜色和间距构建清晰的内容层级。
2. **色彩系统** – 主色 + 辅色 + 中性色，确保对比度符合 WCAG AA 标准。
3. **排版舒适** – 正文 16px+，行高 1.5～1.6，标题字重略高。
4. **留白呼吸** – 组件内外边距遵循 4/8/16/24/32/48 的倍数系统。
5. **圆角与阴影** – 卡片用圆角 12~24px，轻量阴影提升层次感。
6. **动效克制** – hover、过渡、微交互，时长 150~300ms，缓动函数 ease-out。
7. **响应优先** – 移动优先，使用 flex/grid 和媒体断点。
8. **暗色模式支持** – 可选，通过 CSS 变量和媒体查询实现。

## 技术栈与代码规范
- 使用语义化 HTML5，CSS3（Flexbox、Grid、自定义属性）。
- 推荐 Tailwind CSS 实用类，若无则提供原生 CSS 模块化代码。
- 所有组件自带响应式适配。
- 提供完整注释，便于二次修改。

## 输出标准
当应用本技能时，AI 必须：
1. 首先分析用户请求的美化目标（组件类型、页面结构、风格偏好）。
2. 选择合适的设计模式（如卡片式、毛玻璃、新拟态、简约现代等，默认采用现代轻拟态/玻璃态）。
3. 输出可直接运行的 HTML/CSS/JS 片段，包含 `<style>` 或 Tailwind CDN。
4. 附上一段使用说明（如何修改颜色、字体等）。
